Three modes, one idea

A scene goes in. Words come out.

geobard is deliberately small: a thin, well-prompted layer between your map data and a language model. Three endpoints cover three jobs.

👁️

Window view

POST /narrate/window

Stand at the point of view and describe what's around you, in plain human language — near, across the street, off to the left. No coordinates, no map-speak. Pick a detail_level of low, medium, or high.

💬

Ask the scene

POST /narrate/prompt

Pass a question alongside the scene — “What's the quickest way out on foot?” — and get a grounded answer that reasons over what's actually present, staying vague where the data is.

🎨

Image prompt

POST /image/prompt

Turn the same scene into a prompt built for an image model, foregrounding the architecture and city structure so the render is faithful to the place rather than generic.

How it works

One small service, any backend.

geobard speaks the OpenAI chat-completions API, so it runs against whatever you already use. Carefully-tuned prompts do the translation; the model does the writing. Non-standard fields in your GeoJSON are read as extra context.

See it

From feature collection to a paragraph.

The same scene can be set anywhere. Add a system prompt and the voice follows — here, a Forgotten Realms street at midday.

Request · POST /narrate/window 
{
  "geojson": {
    "type": "FeatureCollection",
    "pov":  { "lat": 33.12, "lng": -117.9 },
    "bearing": 110,
    "time_of_day": "12:30",
    "features": [
      { "properties": {
          "type": "street",
          "name": "Selduth Street" } },
      { "properties": {
          "type": "buildings",
          "name": "residential row" } }
    ]
  },
  "detail_level": "medium",
  "system": [
    "You narrate a Forgotten Realms scene."
  ]
}
Response · text

“Looking around, you're standing at the edge of a residential neighborhood where the midday sun beats down on the cobblestones. Selduth Street is just a few paces to your right, narrow and busy, its row houses leaning shoulder to shoulder as it bends out of sight…”

Quickstart

Running in two commands.

Pull the container, point it at a model, and call it. Defaults to OpenRouter; set GEOBARD_OPENAI_BASE_URL for anything else.

run the service
# 1 · start geobard
docker run --rm -p 8000:8000 \
  -e GEOBARD_OPENAI_API_KEY=sk-or-… \
  -e GEOBARD_OPENAI_MODEL=anthropic/claude-haiku-4.5 \
  ghcr.io/openfantasymap/geobard:latest
call an endpoint
# 2 · narrate a scene
curl -sS http://localhost:8000/narrate/window \
  -H "content-type: application/json" \
  -d '{ "geojson": { … },
        "detail_level": "medium" }'

# → { "text": "…", "model": "…" }
API surface

Four routes, that's all.

Every request can override the model and temperature per call. Window and image modes take a detail_level; all narration modes accept optional system prompt lines.

POST/narrate/window
A generic window view. Body takes geojson, optional system[], detail_level. Returns { text, model }.
POST/narrate/prompt
Answer a custom prompt about the scene. Same body plus the question string.
POST/image/prompt
Produce an image-generation prompt. Takes optional image_system[] style hints and a detail_level.
GET/healthz
Returns { "status": "ok" } once required env is present. Used by the Docker healthcheck.
